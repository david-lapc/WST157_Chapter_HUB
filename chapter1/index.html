    Planning a website before designing it

    A strong website starts with purpose, audience, priorities, and decisions the team can defend.

    Opening question What makes a website successful?

    Today’s path

    1. Define strategy in plain language
    2. Separate client wants from user needs
    3. Prioritize features and scope
    4. Build the first project charter

    Use the arrow keys or the Next button to move through the deck.

    Main idea

    Strategy is not decoration.

    A website strategy explains what the site should accomplish, who it serves, and how decisions will stay focused.

    Weak starting point

    “Make it look modern.”

    Stronger starting point

    “Help first-time visitors understand services and request an appointment in under two minutes.”

    Design choices become easier when the project has a clear reason to exist.

    Website planning tension

    The client pays for the site. The users decide whether it works.

    Client wants

    • Beautiful homepage video
    • All services promoted equally
    • Company story near the top
    • Big social media feed

    User needs

    • Hours and location
    • Clear pricing or service options
    • Fast appointment request
    • Proof that the business is trustworthy

    Good planning respects the client and advocates for the user.

    Scope control

    Every “yes” has a cost.

    Balanced first version Enough features to be useful, not so many that the project loses focus. Example launch mix: class schedule, membership pricing, trial booking, testimonials, contact form.

    Prioritizing means removing some good ideas so the strongest ideas can succeed.

    Types of websites

    Different site types need different planning questions.

    Project governance

    Most website problems are decision problems before they are design problems.

    Approves Who can say “yes, this is final”?
    Creates Who writes content, designs screens, and builds pages?
    Maintains Who updates the site after launch?
    Answers Who resolves conflicts when goals compete?

    Clear roles reduce confusion, delays, and scope creep.

    A website is built through connected phases.

    Project charter

    The project charter is the first shared map.

    01Purpose

    Why should this site exist?

    02Goals

    What outcomes should it create?

    03Audience

    Who must the site work for?

    04Success indicators

    How will progress be recognized?

    05Strategies

    What approaches support the goals?

    06Tactics

    What features or actions support the strategies?

    Example

    Project charter snapshot: mobile car wash

    Purpose

    Help busy vehicle owners book mobile detailing with confidence.

    Goals

    Increase booking requests, explain service packages, reduce repeated questions.

    Audience

    Local car owners who value convenience, trust, and clear pricing.

    Success indicators

    More booking form submissions, fewer pricing questions, more repeat customers.

    Strategies

    Make booking easy, prove quality visually, keep package comparison simple.

    Tactics

    Booking button, service cards, before/after gallery, FAQ, map/service area, testimonials.

    Takeaway

    A website plan is a decision-making tool.

    It helps a team know what to build, why it matters, who it serves, and how success will be judged.

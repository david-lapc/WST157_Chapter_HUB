    Chapter 6 · Page Structure

    Design the page before decorating the page

    Page structure is about the reusable regions, semantic order, responsive behavior, and content priority that make individual pages usable.

    Opening questionWhat does every page need before it gets visual style?

    Today’s path

    1. Identify common page regions
    2. Use semantic structure
    3. Plan responsive layouts
    4. Prioritize mobile content
    5. Check headers, main content, sidebars, and footers
    6. Review real websites

    Main idea

    A website is made of pages, and pages need structure.

    Chapter 6 shifts from site maps to page templates. A template creates a consistent system: identity, navigation, main content, supporting content, and footer.

    Site structure

    How pages relate to each other.

    Page structure

    How elements are arranged inside each page.

    Semantic page regions

    Use meaningful containers, not only generic boxes.

    HTML5 supports common page areas like <header>, <nav>, <main>, <article>, <section>, <aside>, and <footer>.

    Good structure helps browsers, screen readers, search engines, developers, and users.

    Source order

    The code order should still make sense without the final layout.

    01Identity

    Logo or site name appears early and links home.

    02Navigation

    Major links are discoverable and keyboard accessible.

    03Main

    The main content is easy to reach and starts with a clear h1.

    04Support

    Related links, side notes, or secondary actions do not interrupt the main task.

    05Footer

    Housekeeping links, contact, copyright, and secondary navigation live at the end.

    06Dates

    Important content should show publication or update context.

    Responsive design

    Do not shrink a desktop page. Re-prioritize the page.

    Responsive pages use fluid layouts, flexible media, and media queries so content adapts to the available viewport.

    The breakpoint should come from the content, not from a random device size.

    Desktop

    • Expanded navigation
    • Multi-column layout
    • Optional side content
    • Large media

    Mobile

    • Prioritized content
    • Compact navigation
    • Single-column stack
    • Performance-aware media

    Mobile-first thinking

    Start with the must-have content and build up.

    1Core taskWhat must the user do?
    2Core contentWhat must appear first?
    3NavigationWhat is essential on small screens?
    4EnhancementWhat can be added on larger screens?

    This keeps the design from becoming a desktop layout squeezed into a phone.

    Page header

    The header is the site’s orientation system.

    IdentityWho owns this page?
    Home linkLogo or site name returns home.
    Global navMajor sections stay visible.
    SearchUseful for larger sites.
    Current stateShows where the user is.

    Convention helps: users expect the logo/home link near the upper left and search often near the upper right.

    Main content

    Do not bury the lead.

    Each page needs a clear visible page title, a strong opening, useful headings, and a structure that supports scanning.

    A page title is not decoration. It confirms the user landed in the right place.

    Content-area checklist

    1. One clear h1
    2. Intro that explains purpose
    3. Headings that preview sections
    4. Links that tell users what happens next
    5. Dates when currency matters
    6. Top/back links for long pages

    Supporting regions

    Sidebars and footers should support—not distract from—the main task.

    Sidebar

    Useful for local navigation, related links, contact details, search, or secondary tools. Keep it consistent.

    Footer

    Useful for copyright, contact, legal links, redundant navigation, related sites, and quiet housekeeping.

    A footer can act like a small sitemap, but it should not become a junk drawer.

    Homepage structure

    The home page is special, but it still belongs to the system.

    The home page introduces the site, clarifies value, surfaces major paths, and directs users to important tasks.

    Chapter 6 suggests designing internal templates first so the home page can introduce the system rather than ignore it.

    Student deliverable

    Create a page-structure plan before visual styling.

    Students should submit a low-fidelity home page or interior page wireframe with labeled regions.

    Must include

    Header, global navigation, main content, one supporting region, footer, mobile priority notes, and one real-site inspiration link.

    Takeaway

    Good page structure makes the site usable before the site becomes beautiful.

    Semantic order, responsive behavior, content priority, and repeatable templates give users the same confidence from page to page.

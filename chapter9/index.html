    Chapter 9 · Typography

    Type is content with structure.

    Typography controls how text is read, scanned, understood, and trusted. On the web, typography also connects visual hierarchy to semantic structure.

    Chapter question: Can users read, scan, and understand the page?
    Opening discussion

    What makes text feel easy or hard to read online?

    Size

    Can the text be read without strain?

    Spacing

    Do lines and paragraphs have enough room?

    Hierarchy

    Can users scan headings and sections?

    Contrast

    Does text separate clearly from the background?

    Typography and meaning

    Typography is visual semantics.

    Headings, lists, paragraphs, captions, quotes, and links are not just styles. They tell people and machines what the content is.

    Looks like a heading

    About the Event

    Made large manually, but still coded like normal text.

    Is a heading

    <h2>About the Event</h2>

    Styled visually and identified structurally.

    Semantic markup

    Choose structure first. Style comes after.

    <h1>The Night Market</h1> <p>Food, art, music, and student creators.</p> <ul> <li>Free entry</li> <li>Friday, 6–10 PM</li> </ul>

    Why it matters

    Semantic markup makes the information hierarchy available to browsers, search engines, assistive technologies, and future stylesheets.

    Do not pick a heading because it “looks big.” Pick it because it is the correct level in the outline.
    Information hierarchy

    A page should have a readable outline.

    H1 · Page topic
    H2 · Major section
    H3 · Subsection
    Paragraph · Explanation, detail, context, support.
    Caption / helper text · Extra information.
    Good hierarchy lets users skim before they commit to reading.
    Body text

    Comfortable reading is designed.

    Featured Vendors

    Browse food, art, music, and student-created products before arriving. Each listing should be short enough to scan and clear enough to help visitors decide what they want to see first.

    Readable type depends on size, line length, line height, contrast, and spacing around paragraphs.

    Display type

    Display type creates landmarks.

    Without landmarks

    Long text blocks become a wall. Users cannot quickly see sections, actions, or priority.

    With landmarks

    Headlines, subheads, pull quotes, labels, and buttons create a path through the content.

    Display type should guide the scan, not compete with every other element.
    White space and alignment

    Margins and spacing are part of typography.

    Crowded

    The content feels harder to scan because the heading, paragraph, and buttons are packed too closely together.

    Readable

    The content feels calmer because spacing creates clear relationships between heading, paragraph, and action.

    Spacing tells the eye what belongs together and what is separate.
    Emphasis

    If everything is emphasized, nothing is emphasized.

    Emphasis rule

    Choose one emphasis method and use it with restraint.

    Links and color

    A link should look like something that can be clicked.

    Weak signal

    Click here to see the schedule.

    Color alone may not be enough.

    Clear signal

    View the full event schedule

    Meaningful link text + recognizable link styling.

    Do not use underline for decoration if users may read it as a link.
    Responsive typography

    Text has to survive zoom, phones, and user preferences.

    Use flexible thinking: relative units, readable line length, scalable spacing, and layouts that adapt when text gets larger.

    Scales
    Wraps
    Reflows
    Stays readable
    A design that only works at one exact size is fragile.
    Type system

    Create repeatable type styles.

    A type scale makes the whole site feel intentional.
    Typeface choice

    Fonts are chosen for context, not just personality.

    Legibility

    Can people read it comfortably at the sizes where it will actually appear?

    Tone

    Does the typeface fit the message: serious, playful, premium, urgent, calm?

    System

    Does it work as a family of headings, body text, buttons, captions, and labels?

    A beautiful font can still be the wrong font.

    Chapter 4 · Information Architecture

    Making information findable

    Information architecture is the structure behind a website: the categories, labels, pages, paths, and relationships that help people find what they need.

    Opening question Where would a visitor click first?

    Today’s path

    1. Inventory what the site has and what it needs
    2. Group content into useful categories
    3. Name things consistently
    4. Choose a structure that matches user expectations
    5. Sketch diagrams and wireframes before visual design
    6. Test the organization with real tasks

    Use the arrow keys or the Next button to move through the deck.

    Main idea

    Visual design cannot rescue confusing structure.

    A site can look polished and still fail if the content is hard to understand, hard to browse, or labeled in a way users do not expect.

    Surface design

    Colors, type, images, spacing, motion, and visual style.

    Information architecture

    Pages, categories, labels, navigation, relationships, diagrams, metadata, and task paths.

    Information architecture decides how the site makes sense before it decides how the site looks.

    What IA does

    Information architecture turns a pile of content into a usable system.

    01Organize

    Group content into taxonomies, sections, and hierarchies.

    02Label

    Use consistent names so similar things are called the same thing.

    03Navigate

    Plan paths that help people move through the site.

    04Map

    Communicate the structure with outlines, diagrams, and wireframes.

    05Test

    Check whether the organization matches real user expectations.

    06Maintain

    Plan standards for content, metadata, accessibility, search, and updates.

    Five-step workflow

    A practical path for organizing site content.

    1InventoryWhat content exists? What content is missing?
    2OutlineWhat are the major categories and relationships?
    3ChunkHow can long information become modular, scannable pieces?
    4DiagramHow will the site and page structures be shown?
    5TestCan real users find things through the structure?

    The goal is not to make a perfect first map. The goal is to make a structure that can be discussed, tested, and improved.

    • RSVP
    • Get Tickets

    Consistent system

    • Use one main label for the main action
    • Use supporting copy to clarify details
    • Keep navigation, buttons, and page titles aligned
    • Choose labels users understand, not just internal terms

    Ways to organize information

    The same content can be organized in different ways.

    CategoryBy similarity or topic
    TimeBy sequence, schedule, or timeline
    LocationBy place, map, area, or region
    AlphabeticBy name, A–Z lists, dictionary, directory
    ContinuumBy price, rating, size, difficulty, or score

    A useful organization scheme matches how the user thinks about the task.

    Segmenting information

    Most web content should be chunked for scanning.

    Users often come to a website with a specific question. Headings, short sections, lists, and consistent patterns help them find the relevant piece faster.

    Chunking is not just shorter writing. It is content divided into useful, reusable pieces.

    Unchunked

    The event has many things happening and visitors should know about the food, location, parking, schedule, vendor rules, ticket policies, and contact information before they arrive, especially because some activities change depending on time and weather.

    Chunked

    • When: Saturday, 6–10 PM
    • Where: Main courtyard
    • Bring: Student ID and payment card
    • Plan ahead: Parking fills quickly

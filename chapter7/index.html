    Main idea

    The interface is the frame, not the painting.

    Good interface design should help users move, choose, submit, search, filter, recover, and stay oriented without pulling attention away from the content.

    Interface asks

    What can I click, choose, open, close, submit, or change?

    Wayfinding asks

    Where am I, where can I go, and how do I get back?

    Principle 1

    No page should trap the user.

    Users may arrive from search, links, social media, or bookmarks. They may never see the homepage first.

    Every page needs a way back to home and a way into the main sections of the site.

    Principle 2

    Real content should not be buried behind too many menu layers.

    1Too shallowToo many choices at once.
    2Too deepToo many clicks before content.
    3BalancedClear categories with real content close by.

    Menus should expose popular destinations, not just act as a transportation system.

    Principle 3

    Familiar usually beats clever.

    Risky interface

    Unusual metaphors, mystery icons, hidden navigation, inconsistent button styles, and labels users must decode.

    Usable interface

    Familiar controls, clear labels, visible states, predictable locations, and consistent patterns across the site.

    Creativity should support the task, not become the task.

    Navigation modes

    Users browse and search, often in the same session.

    Search helps users who know what they want. Browse navigation teaches users what the site contains and what words the site uses.

    Search does not replace good navigation.

    Browse supports

    • Discoverability
    • Context
    • Major categories
    • Orientation

    Search supports

    • Known-item finding
    • Specific queries
    • Large content sets
    • Direct access

    Wayfinding

    The interface should answer: where am I?

    Site identityWho owns this page?
    Page titleWhat page am I on?
    Selected navWhich section is active?
    BreadcrumbsHow does this page fit?
    Footer linksWhere can I recover or continue?

    Search can drop users deep inside a site, so page-level cues matter.

    Navigation systems

    Headers create global wayfinding.

    A strong header usually contains site identity, a home link, major navigation, and sometimes search or account tools.

    Navigation should reveal what the site offers, not just provide links.

    Header critique checklist

    1. Is the logo/name clear?
    2. Does the logo link home?
    3. Are major sections visible?
    4. Is the current section indicated?
    5. Is search easy to find?
    6. Does the mobile version still work?

    Menus, links, and buttons

    Controls should look like what they do.

    01Links

    Move to another page, document, or location.

    02Buttons

    Trigger an action: submit, save, open, close, continue.

    03Menus

    Show a set of available choices.

    04Tabs

    Switch between related panels or sections.

    05Breadcrumbs

    Show hierarchy and provide a recovery path.

    06Pagination

    Moves through sequences or result sets.

    Forms

    Forms are conversations with constraints.

    Use fields when users need to enter open-ended information. Use menus, checkboxes, radios, and autocomplete when the possible answers can be guided.

    Every form should make labels, required fields, errors, and next steps obvious.

    Good form behavior

    • Clear labels
    • Useful defaults
    • Helpful examples
    • Inline errors
    • Visible submit action

    Common problems

    • Placeholder-only labels
    • Vague errors
    • Too many required fields
    • Unclear success state
    • Keyboard traps

    Interactive components

    Hidden content needs visible controls and states.

    Useful toggles

    Accordions, FAQ panels, drawers, filters, show/hide details, and expandable outlines can simplify complex content.

    Accessibility check

    Can the control be focused, named, operated by keyboard, understood by screen readers, and visually recognized when open or closed?

    Use native HTML controls when possible before creating custom controls.

    Mobile interface design

    Small screens change interaction, not just layout.

    1Viewport

    Users see only a small portion at a time.

    2Touch

    Targets need room and clear feedback.

    3Context

    Users may be distracted, moving, or on weak connections.

    4Priority

    Important actions must not be buried.

    5Input

    Typing is harder; simplify forms.

    6Continuity

    Mobile users still want the full experience.

    Activity

    Interface critique: find the next step.

    Choose one real website. Start on an interior page, not the homepage. Answer: Where am I? What can I do? What is the main action? How do I recover if I am lost?

    Which element belongs in the main content area?

    Student deliverable

    Create an interface checklist for your project.

    Students should identify the navigation, controls, forms, feedback states, search/browse support, and mobile interface decisions for their site.

    Takeaway

    Interface design helps users act with confidence.

    A good interface makes location, options, actions, feedback, and recovery visible enough that users do not have to guess.
